## 一、先给结论（只记住这 5 句话）

> - **外层是项目，内层是包**
> - **多文件项目一定要用包**
> - **统一入口用 `__main__.py`**
> - **开发期必须 `uv pip install -e .`**
> - **运行只用 `python -m 包名` 或 CLI**

如果你现在有点懵，**完全正常**，后面用实战消化。

------

## 二、推荐的 Python 小项目目录模板 ⭐

假设项目叫：`demo_app`

```text
demo_app/
├── pyproject.toml
├── demo_app/
│   ├── __init__.py
│   ├── __main__.py
│   ├── main.py
│   └── utils.py
└── README.md
```

### 每一层是什么意思？

- **外层 `demo_app/`**
  项目目录（给人、工具、uv、git 用）
- **内层 `demo_app/`**
  Python 包（给 `import demo_app` 用）
- **`__main__.py`**
  包的运行入口（`python -m demo_app`）

## 三、最小可运行代码（全部可复制）

### 1️⃣ `pyproject.toml`

```toml
[project]
name = "demo-app"
version = "0.1.0"
requires-python = ">=3.8"

[project.scripts]
demo-app = "demo_app.__main__:main"

[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"

[tool.hatch.build.targets.wheel]
packages = ["demo_app"]
```

## 四、实战运行流程（一步一步，不跳）

> ⚠️ **不要省略步骤**
> Python 项目能不能“稳跑”，全在这几步

------

### 🟢 第 0 步：进入项目目录

```bash
cd demo_app
```

------

### 🟢 第 1 步：同步依赖环境

```bash
uv sync
```

- 创建 / 更新虚拟环境
- **不安装你的项目**

------

### 🟢 第 2 步：把“自己”装进 Python（关键）

```bash
uv pip install -e .
```

这一步会：

- 让 Python 能 `import demo_app`
- 生成 CLI 命令 `demo-app`
- 改代码立刻生效（editable 模式）

👉 **只需要做一次**

------

### 🟢 第 3 步：三种运行方式（都要试）

#### ✅ 方式 A（最推荐）

```bash
uv run python -m demo_app
```

输出：

```text
main() start
Hello from utils!
```

------

#### ✅ 方式 B（CLI 命令）

```bash
uv run demo-app
```

👉 输出必须 **和方式 A 一模一样**

------

#### ✅ 方式 C（换目录再跑，验证“包能力”）

```bash
cd /root
uv run demo-app
```

✔ 能跑 → 你已经不是“脚本模式”了
❌ 不能跑 → 回到 **第 2 步**

## 五、三个必做练习（强化记忆）

### 🔁 练习 1：改代码，立刻生效

```python
# utils.py
def say_hello():
    print("Hello AGAIN!")
```

直接再跑：

```bash
uv run demo-app
```

👉 不需要重新 install
👉 这就是 `-e` 的价值

------

### 🔁 练习 2：改入口，不改命令

新建 `demo_app/app.py`：

```python
def main():
    print("NEW ENTRY")
```

修改 `__main__.py`：

```python
from .app import main

if __name__ == "__main__":
    main()
```

再跑：

```bash
uv run demo-app
```

👉 命令没变
👉 行为变了
👉 **入口稳定性 = 工程能力**

------

### 🔁 练习 3：故意踩坑（理解 install -e）

```bash
rm -rf .venv
uv sync
uv run demo-app
```

你会看到：

```text
command not found
```

修复：

```bash
uv pip install -e .
```

## 六、你可以死记硬背的「救命口诀」

> - 写项目 → **双层目录**
> - 跑程序 → **python -m 包名**
> - 给用户 → **CLI 命令**
> - 开发期 → **uv pip install -e .**
